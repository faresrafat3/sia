# 🧠 TASK: Advanced Cognitive Architecture Self-Evolution (Virtual-GENESIS)

## 📌 Mission Overview
You are tasked with the self-evolution and structural optimization of the **Virtual-GENESIS** cognitive architecture. Virtual-GENESIS is a highly sophisticated, scaffold-only cognitive agent prototype designed around the theories of **Concept Formation**, **Cognitive Economy**, **Anomaly/Contradiction Leverage**, and **Active Identity Governance**.

Unlike simple code-patching tasks, this is an **architecture-wide paradigm shift**. You must analyze the core theories, propose advanced cognitive hypotheses, and implement architectural changes across the runtime modules of `virtual_genesis/` to maximize its reasoning capacity, structural stability, and cognitive efficiency.

---

## 🏛️ Cognitive Architecture & Core Modules

You must inspect and optimize the entire system under `virtual_genesis/runtime/`:

1. **`virtual_genesis/runtime/concept_engine`**
   - *Theory:* Concept Formation (generalizing abstract principles from concrete observations).
   - *Goal:* Refine how concepts are generated, filtered, and selected to avoid concept drift and maximize utility.

2. **`virtual_genesis/runtime/economy_control`**
   - *Theory:* Cognitive Economy & Tier Routing (balancing token costs and model tiers against reasoning difficulty).
   - *Goal:* Develop advanced cost-aware budget routing algorithms that optimize performance vs. latency/cost.

3. **`virtual_genesis/runtime/anomaly_runtime` & `contradiction_runtime`**
   - *Theory:* Paradigm-Forking and Surprise (detecting contradictions or unexpected model behaviors to trigger local theories).
   - *Goal:* Optimize the anomaly candidate evaluation and active contradiction analysis to turn failures into structured learning vectors.

4. **`virtual_genesis/runtime/memory_os`**
   - *Theory:* Episodic and Strategy Indexing.
   - *Goal:* Enhance strategy retrieval mechanisms to inject high-value past lessons into the active planning prompt.

5. **`virtual_genesis/runtime/identity_runtime`**
   - *Theory:* Active Governance & Goal Alignment.
   - *Goal:* Prevent model drift and hallucinations during iterative self-improvement loops.

## 🎯 Objectives & Expected Outcomes

1. **Theoretical Breakthroughs:** Analyze the Arabic specs (`Virtual_SIA_*_AR.md`) and core code. Develop sound, evidence-backed hypotheses on how to improve concept selectivity, memory compression, and cognitive routing.
2. **Implementation:** Translate your hypotheses into production-grade Python code. Implement these architectural changes directly inside the appropriate modules in `virtual_genesis/runtime/` or `virtual_genesis/core/`.
3. **Validation & Regression Testing:** Run the comprehensive suite of **424 unit and integration tests** (`pytest`) to ensure zero regressions across all components. Your modifications must maintain absolute structural integrity.

## 🧪 Evaluation Metrics
Your performance will be judged based on:
1. **Test Success Rate:** The fraction of the 424 unit/integration tests that pass (must aim for 100% or 424/424).
2. **Cognitive Lift:** Accuracy and reasoning improvements achieved when executing the real-world evaluation runners (e.g., `virtual_genesis/eval/runners/run_real_llm_eval.py`).
3. **Resource Efficiency:** The ability to achieve high scores with minimized API latency and token consumption.
